## 二、参考开源框架相关信息（核心，必提供）

### 开源项目全称及官方地址

- **名称**：Zotero-arXiv-Daily  
- **地址**：<https://github.com/TideDra/zotero-arxiv-daily>

### 开源项目的核心功能、整体框架（简要）

根据上游 README：根据用户 Zotero 书库，从 **arXiv**获取新稿，用**嵌入向量**计算与书库的相似度并加权排序，调用 **LLM 生成 TL;DR**，将结果通过邮件推送；典型部署为 **GitHub Actions** 定时任务，也可本地 `uv run` 执行。

### 参考该开源框架的具体内容

参考其**业务闭环**：Zotero 拉取语料 → 外部文献源检索 → 基于嵌入的相似度打分与排序 → LLM 生成短摘要 → SMTP 邮件推送；以及**配置驱动**（环境变量 + YAML）的思想。本仓库为**独立实现**（目录结构、模块划分、依赖选型与上游不一致），并非 fork 后小改。

### 开源项目的不足 / 改进初衷（结合毕设）

1. 上游以 **GitHub Actions 托管**为主，与本课题强调的**本地化部署、统一配置与实验复现**需求不完全一致。  
2. 上游 README 中算法说明相对简略，且承认推荐较简单；毕设需要**更可解释、可扩展**的排序与展示。  
3. 课题要求突出「大模型」与「推送系统」完整度，需要**可视化配置、可选反馈闭环、多源（含学术索引 OpenAlex）** 等能力，故在参考思路基础上重新设计与实现。
4. 仅实现了从 **arXiv**获取文献，文献来源不足。
5. 默认拉取符合的所有文献，没有针对日期进行筛选
6. 每次都进行

## 三、项目改进及创新点（核心，必提供）

### 具体改进内容（原框架问题 + 改进方案）

1. **文献来源与工程组织**  
   - **原**：侧重 arXiv 生态与 Actions 工作流。  
   - **改**：采用 **Hydra + `config/base.yaml` / `custom.yaml`**；增加 OpenAlex，bioRxiv/medRxiv 检索；支持 `source_error_policy`（单源失败时继续或快速失败）。

2. **排序与可解释性**  
   - **原**：加权平均相似度，解释性较弱。  
   - **改**：**OpenAI 兼容嵌入 API** 批量向量、余弦相似度 + **书库加入时间衰减**；邮件中输出 **「为什么推荐」**（关键词命中 + 书库条目贡献：余弦、时间权重、贡献值）。

3. **兴趣表达与展示**  
   - **原**：主要依赖嵌入匹配。  
   - **改**：从书库摘要抽取 **TF-IDF 关键词**（`keywords` 模块），用于展示与候选稿上的命中说明。

4. **大模型与摘要输入**  
   - **原**：TL;DR 侧有从 PDF 抽文本等路径（上游 README 提及 pymupdf4llm）。  
   - **改**：以元数据 **标题/摘要** 为输入生成 TL;DR，与嵌入、邮件渲染在同一流水线中串联（`executor`）。

5. **配置与使用体验**  
   - **原**：以仓库 Secrets / Variables 为主。  
   - **改**：**Streamlit** 页面（`web_app.py`）读写 `custom.yaml`，降低配置成本，实现可视化控制。

6. **反馈闭环（可选）**  
   - **原**：无本项目中同类独立反馈服务。  
   - **改**：邮件内 **签名反馈链接**，独立 HTTP 服务（`feedback_server`）、**SQLite**、**HMAC + TTL**，并对后续排序做 **反馈加权**（`feedback` 模块）。

### 改进的技术实现细节（简要）

- 配置：`Hydra` + `OmegaConf`，环境变量与 YAML 合并。  
- 检索：`retriever/` 各源实现统一协议，由 `Executor` 聚合。  
- 重排：`reranker.api` 批量请求嵌入 API；**书库向量磁盘缓存**（`cache_store`）减少重复调用。  
- 报告：`render_markdown` / `render_html` 双格式邮件正文。  
- 测试：`pytest` 覆盖检索、重排、缓存、反馈、报告等（`tests/`）。

### 创新点（1～2 个）

1. **可解释的个性化重排**：在向量相似度与时间衰减之外，向用户展示**关键词命中**与**具体书库文献对总得分的贡献**，便于理解与调参。  
2. **「追踪—推送—反馈—再排序」闭环**：在邮件场景下用签名链接与本地存储实现**轻量闭环**，区别于仅单次推送的日报类工具。

### 改进效果

- **功能**：在 arXiv 日报之外，增加 **OpenAlex**、**bioRxiv/medRxiv**、**Web 配置**、**推荐理由分解**、**可选反馈闭环**。
